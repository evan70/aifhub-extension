# Архитектура: Skill-Based Extension

## Обзор

Это расширение для ai-factory 2.x использует **Skill-Based Architecture** — коллекцию независимых навыков (skills), каждый из которых является самодостаточным модулем с собственной инструкцией и ресурсами. Навыки не имеют программных зависимостей между собой и взаимодействуют только через файловую систему (чтение/запись артефактов).

## Обоснование выбора

- **Тип проекта:** Extension (расширение для ai-factory CLI)
- **Технологии:** Markdown-документы (SKILL.md) + references
- **Ключевой фактор:** Навыки должны быть переносимыми, независимыми и легко устанавливаемыми в любой проект

## Структура папок

```
aifhub-extension/
├── extension.json              # Манифест расширения (имя, версия, список skills)
│
├── skills/                     # Коллекция навыков
│   ├── aif-analyze/            # Навык анализа проекта
│   │   ├── SKILL.md            # Главная инструкция навыка
│   │   └── references/         # Вспомогательные ресурсы
│   │       ├── description-template.md
│   │       └── project-scan-checklist.md
│   │
│   └── aif-roadmap-plus/       # Навык создания roadmap
│       ├── SKILL.md
│       └── references/
│           ├── roadmap-template.md
│           └── slice-checklist.md
│
├── .github/
│   └── workflows/
│       └── tag-version.yml     # CI: автотегирование при изменении версии
│
├── .ai-factory/                # AI-контекст проекта
│   ├── DESCRIPTION.md          # Описание проекта
│   ├── ARCHITECTURE.md         # Этот файл
│   └── ROADMAP.md              # Roadmap проекта
│
├── AGENTS.md                   # Карта проекта для AI-агентов
└── README.md                   # Документация для пользователей
```

## Правила зависимостей

### ✅ Разрешённые зависимости

```
SKILL.md → references/*        (навык использует свои шаблоны)
skill A → skill B output        (чтение артефактов, созданных другим навыком)
```

### ❌ Запрещённые зависимости

```
skill A → skill B internal      (прямое обращение к внутренней логике другого навыка)
references/* → SKILL.md         (ресурсы не должны зависеть от инструкции)
```

## Коммуникация между навыками

Навыки взаимодействуют через артефакты в файловой системе:

| Навык | Читает | Пишет |
|-------|--------|-------|
| `aif-analyze` | `AGENTS.md`, `.ai-factory/RULES.md` | `.ai-factory/DESCRIPTION.md` |
| `aif-roadmap-plus` | `.ai-factory/DESCRIPTION.md`, `.ai-factory/ARCHITECTURE.md` | `.ai-factory/ROADMAP.md` |

## Ключевые принципы

1. **Независимость навыков** — Каждый навык полностью самодостаточен и может работать изолированно
2. **YAML frontmatter** — Каждый `SKILL.md` начинается с `name` и `description` для регистрации в ai-factory
3. **References как ресурсы** — Шаблоны и чеклисты в `references/` — это данные, не код
4. **Evidence-based** — Навыки анализируют код, а не полагаются на документацию

## Пример структуры навыка

### SKILL.md с YAML frontmatter

```markdown
---
name: aif-analyze
description: Краткое описание для ai-factory
---

# Название навыка

Инструкция для AI-агента...

## Workflow
1. Шаг 1
2. Шаг 2

## Rules
- Правило 1
- Правило 2
```

### References файл (template)

```markdown
# Template Title

Шаблон или чеклист для использования в SKILL.md...
```

## Анти-паттерны

- ❌ **Не дублируйте логику между навыками** — Если общая логика нужна, вынесите в отдельный файл или навык
- ❌ **Не создавайте циклические зависимости** — Навык A не должен требовать артефакт, который создаёт навык B, если B требует артефакт от A
- ❌ **Не хардкодьте пути** — Используйте переменные или соглашения (`.ai-factory/`, `AGENTS.md`)
- ❌ **Не смешивайте инструкции и данные** — SKILL.md — это логика, references/ — это данные

## Стратегия добавления новых навыков

1. Создать папку `skills/<название>/`
2. Добавить `SKILL.md` с YAML frontmatter (name, description)
3. Добавить необходимые файлы в `references/`
4. Зарегистрировать навык в `extension.json` → `skills[]`
5. Если навык заменяет существующий — добавить в `extension.json` → `replaces`
